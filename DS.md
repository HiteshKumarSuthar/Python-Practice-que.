# Complete Data Science Roadmap — 2nd Year AI&DS Student

**Target:** Strong DS fundamentals + Internship by 2nd year end + Big product company readiness (with SDE option open)

---

## PHASE 1: Foundations (Month 1-3)

### 1. Python (Ongoing — already started)
- Basics: variables, loops, functions, OOP concepts
- Data structures: lists, dictionaries, tuples, sets
- File handling, exception handling
- **Resource:** freeCodeCamp Python course, or W3Schools for quick reference

### 2. DSA (Start after 10-15 days, Python-based)
- Arrays, Strings, Searching, Sorting
- Linked List, Stack, Queue
- Recursion & Backtracking
- Trees, Graphs
- Dynamic Programming
- **Resource:** Striver's A2Z DSA Course
- **Target:** 150-200 problems by Month 6

### 3. Git & GitHub
- Basic commands: init, add, commit, push, pull
- Create repos, push your Python/DSA code regularly
- **Time needed:** 1-2 days to learn basics

---

## PHASE 2: Data Handling Skills (Month 4-6, parallel with DSA)

### 4. SQL
- SELECT, WHERE, ORDER BY, GROUP BY
- JOINs (inner, left, right, full)
- Subqueries, Window Functions
- **Resource:** HackerRank SQL, Mode Analytics SQL Tutorial

### 5. NumPy
- Arrays, array operations, broadcasting
- Mathematical operations on arrays

### 6. Pandas
- DataFrames, Series
- Data cleaning: handling missing values, duplicates
- Data manipulation: filtering, grouping, merging
- **Resource:** Kaggle Learn "Pandas" course

### 7. Data Visualization
- Matplotlib basics: line plots, bar charts, histograms
- Seaborn: better-looking statistical plots
- **Resource:** Kaggle Learn "Data Visualization" course

---

## PHASE 3: Statistical & ML Foundations (Month 6-8)

### 8. Statistics & Probability
- Mean, median, mode, variance, standard deviation
- Probability basics, distributions (normal, binomial)
- Hypothesis testing, p-value, confidence intervals
- Correlation vs Causation
- **Resource:** StatQuest (YouTube) — explains everything simply

### 9. Machine Learning Basics
- Supervised vs Unsupervised learning
- Linear Regression, Logistic Regression
- Decision Trees, KNN, basic clustering (K-Means)
- Model evaluation: accuracy, precision, recall, MSE, R²
- **Resource:** Kaggle Learn "Intro to Machine Learning", Andrew Ng's Coursera course

### 10. Scikit-learn
- Implementing ML algorithms practically
- Train-test split, model fitting, prediction, evaluation

---

## PHASE 4: Practical Application (Month 7-9)

### 11. Kaggle Practice
- Create account, do beginner competitions (Titanic dataset)
- Read top-ranked notebooks to learn techniques
- Publish your own notebook

### 12. Build 2-3 Projects
- **Project 1 (Beginner):** Titanic Survival Prediction or House Price Prediction
- **Project 2 (Intermediate):** Customer Churn Prediction or Sentiment Analysis
- **Project 3 (Optional):** Sales Forecasting or Recommendation System
- Each project must include: Data cleaning → EDA → Model building → Evaluation → GitHub upload with README

### 13. Google Colab
- Use for running Python/ML code without local setup
- Free GPU access for slightly heavier models

---

## PHASE 5: Internship Prep (Month 9-10)

### 14. Resume Building
- Skills section: Python, SQL, Pandas, NumPy, ML basics
- Projects with GitHub links
- Kaggle profile link (if you have notebooks/competitions)

### 15. Interview Prep
- DSA: Easy-Medium LeetCode problems (100-150 solved)
- SQL: Practice interview-style queries
- ML concepts: Be ready to explain your projects in depth
- Basic CS fundamentals: OOPs concepts overview

### 16. Where to Apply
- LinkedIn ("Data Science Intern" search)
- Internshala
- Company career pages (startups often direct-apply)
- College placement cell

## Golden Rules to Follow

1. **Max 2 things parallel** — don't try to learn everything at once (e.g., DSA + SQL together is fine, adding ML on top will overwhelm)
2. **Depth over breadth** — better to know Pandas well than to have touched 10 tools superficially
3. **Build as you learn** — don't wait to "finish learning" before starting projects; start a small project after Pandas + basic ML
4. **Consistency > Intensity** — 1-2 hours daily beats 10 hours once a week
5. **Push code to GitHub regularly** — build the habit from day one, even for small scripts
